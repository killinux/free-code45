  • 内存管理系统 ▼
    • 微压缩 (Micro Compact) ▼
      • 时间触发 ▼
        • 间隔 ≥ 60 分钟时触发（服务端缓存已过期）
        • 直接修改消息，替换为 [Old tool result content cleared]
        • 保留最近 5 个工具结果（最少 1 个）
      • 缓存编辑 (Cached MC) ▼
        • 工具结果数 ≥ 12 时触发
        • 通过 cache_edits API 服务端删除，不修改本地
        • 保留最近 3 个结果
        • 支持模型：opus-4-6, sonnet-4-6
      • 可压缩工具：FileRead / Bash / Grep / Glob / WebSearch / WebFetch / FileEdit / FileWrite
    • 自动压缩 (Auto Compact) ▼
      • 触发阈值：有效窗口 - 13K（约 93% 占用率）
      • 阻塞限制：有效窗口 - 3K（约 98%）
      • 熔断器：连续 3 次失败后停止重试
      • 优先级 1: SM Compact（零 API） ▼
        • 使用已提取的会话笔记作为摘要
        • 保留 10K-40K token 的近期消息
        • 最少 5 条文本消息
        • adjustIndexToPreserveAPIInvariants() 保护配对
      • 优先级 2: 传统压缩（1 次 API） ▼
        • 发送完整对话给 LLM 生成 9 章节摘要
        • 摘要最大输出 20K token
        • 压缩后恢复 CLAUDE.md + 技能（50K token 预算）
    • 会话记忆 (Session Memory) ▼
      • 初始化阈值：10,000 token
      • 更新增量阈值：5,000 token（必要条件）
      • 工具调用阈值：3 次
      • 12 个分区，每分区 2K token，总计 12K
      • 通过 runForkedAgent() 分叉子代理执行
      • 子代理仅允许 FileEdit 编辑指定记忆文件
      • 存储：~/.claude/session-memory/MEMORY.md
    • 持久记忆 (Auto Memory) ▼
      • 触发：每轮结束时（最后一条回复无工具调用）
      • 节流：每 N 轮执行一次（默认 1）
      • 最大轮次：5（防止验证兔子洞）
      • 互斥：主代理写入时跳过后台提取
      • 重叠：暂存上下文，完成后执行一次尾随提取
      • 权限沙箱 ▼
        • Read / Grep / Glob — 不限制
        • Bash — 仅只读命令
        • Edit / Write — 仅 memory 目录
        • 其他工具 — 全部拒绝
      • 4 种记忆类型：user / feedback / project / reference
      • 两步保存：写文件 + 更新 MEMORY.md 索引
    • 记忆合并 (Auto Dream) ▼
      • 时间门控：≥ 24 小时
      • 会话门控：≥ 5 个新会话
      • 锁机制：原子文件 mtime 更新
      • 扫描节流：10 分钟
    • 记忆注入 (Injection) ▼
      • 扫描 → 调 Sonnet 3.5 选出 ≤5 个相关文件
      • 每文件 4KB，每轮 ~20KB，会话 60KB
      • 非阻塞预取（永不阻塞主轮次）
      • >1 天的记忆附带新鲜度警告
      • 压缩后重置累计计数器
    • 团队记忆安全 ▼
      • 路径遍历防护：null 字节 / URL 编码 / Unicode / 反斜杠
      • 两阶段验证：字符串级 + 符号链接解析
      • 前缀攻击防护：要求分隔符紧跟前缀
      • 禁止保存：API 密钥、凭证等敏感数据
  • 上下文管理系统 ▼
    • 窗口配置 ▼
      • 默认窗口：200K，1M 模型：1,000,000
      • Opus 4.6 输出：64K/128K
      • Sonnet 4.6 输出：32K/128K
      • 槽位优化：初始 8K，重试升级到 64K
      • 环境变量覆盖：CLAUDE_CODE_MAX_CONTEXT_TOKENS
    • 8 阶段处理管线 ▼
      • 1. 边界提取 — getMessagesAfterCompactBoundary()
      • 2. 预算截断 — applyToolResultBudget()
      • 3. Snip 裁剪 — 删除旧的中间轮次
      • 4. 微压缩 — 清理工具结果（时间/缓存编辑）
      • 5. 上下文折叠 — 细粒度摘要化（实验性）
      • 6. 自动压缩 — SM 压缩或传统压缩
      • 7. 阻塞检查 — ~98% 时阻止输入
      • 8. 组装请求 — 规范化 + 拼接 + 缓存控制
    • Prompt 缓存 ▼
      • 系统提示词分割：静态前缀（缓存） + 动态后缀
      • 默认 TTL 5 分钟，实验性 1 小时
      • 缓存中断检测：跟踪 15+ 状态参数
      • 状态锁存：Beta / TTL / 努力等级 会话内不变
    • 消息规范化 ▼
      • 合并同源记录（同一 message.id）
      • 剥离 advisor 块、caller 字段、tool_reference 块
      • ensureToolResultPairing() 确保工具配对
      • API 轮次分组：以 message.id 变化为界
    • 错误恢复 ▼
      • prompt_too_long → 折叠恢复 / 响应式压缩
      • max_output_tokens → 升级到 64K 重试
      • 媒体错误 → 移除图片/PDF，文本替代后重试
