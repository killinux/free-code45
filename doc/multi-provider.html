Claude Code 多供应商配置

CLAUDE_CONFIG_DIR 一键切换不同 API 供应商
快速开始 原理 配置示例 搭建步骤 Alias 设置 同时运行 settings.json 参考 常见问题

快速开始

一句话：每个供应商一个配置目录，通过 CLAUDE_CONFIG_DIR 环境变量切换。零代码改动，立即可用。 
# 默认供应商（使用 ~/.claude/）
./cli

# 切换到 Kimi
CLAUDE_CONFIG_DIR=~/.claude-kimi ./cli

# 切换到 MiniMax
CLAUDE_CONFIG_DIR=~/.claude-minimax ./cli

原理

Claude Code 的所有配置（API Key、Base URL、Model、会话、记忆等）都存储在 ~/.claude/ 目录下。 全局路径通过 getClaudeConfigHomeDir() 解析，该函数尊重 CLAUDE_CONFIG_DIR 环境变量。

CLAUDE_CONFIG_DIR 未设置: getClaudeConfigHomeDir() → ~/.claude/ CLAUDE_CONFIG_DIR=~/.claude-kimi: getClaudeConfigHomeDir() → ~/.claude-kimi/

设置不同的 CLAUDE_CONFIG_DIR 值，两个实例使用完全独立的配置目录，互不干扰。

~/.claude/ ← 默认 (MiniMax) ├── settings.json ← MiniMax 的 API Key / Base URL / Model ├── .claude.json ← MiniMax 的全局配置 ├── projects/ ← MiniMax 的会话历史 └── ... ~/.claude-kimi/ ← Kimi ├── settings.json ← Kimi 的 API Key / Base URL / Model ├── .claude.json ← Kimi 的全局配置 (首次启动自动生成) ├── projects/ ← Kimi 的会话历史 └── ...

Alias 设置

将以下内容加入 ~/.bashrc 或 ~/.zshrc：

# Claude Code 多供应商 alias
CLI_PATH="/opt/workspace/myclaude/free-code45/cli"

alias claude-minimax="$CLI_PATH"
alias claude-kimi="CLAUDE_CONFIG_DIR=~/.claude-kimi $CLI_PATH"

# 如有更多供应商，继续添加：
# alias claude-deepseek="CLAUDE_CONFIG_DIR=~/.claude-deepseek $CLI_PATH"
# alias claude-openai="CLAUDE_CONFIG_DIR=~/.claude-openai $CLI_PATH"
# 生效
source ~/.bashrc

# 使用
claude-minimax        # MiniMax
claude-kimi           # Kimi

同时运行多个供应商

完全安全。每个实例操作自己独立的 lockfile、PID 文件和配置文件，互不干扰。 
# 终端 1 — MiniMax
claude-minimax

# 终端 2 — Kimi (同时运行)
claude-kimi

字段说明示例

env.ANTHROPIC_BASE_URL API 端点地址 https://api.moonshot.cn/anthropic
env.ANTHROPIC_MODEL 主模型名 kimi-k2.5
env.ANTHROPIC_DEFAULT_OPUS_MODEL Opus 级别模型映射 kimi-k2.5
env.ANTHROPIC_DEFAULT_SONNET_MODEL Sonnet 级别模型映射 kimi-k2.5
env.ANTHROPIC_DEFAULT_HAIKU_MODEL Haiku 级别模型映射 kimi-k2.5
env.ANTHROPIC_SMALL_FAST_MODEL 轻量快速模型（工具搜索等） MiniMax-M2.5
env.API_TIMEOUT_MS API 超时时间（毫秒） 600000 (10 分钟)
env.CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC 禁用非必要网络请求（遥测、更新检查等） "1"
model 顶层模型设置（可选，覆盖 env 中的值） "kimi-k2.5"
permissions 工具权限规则 见下方说明

权限配置说明

"permissions": {
  "allow": ["Read(*)", "Glob(*)"],          // 自动允许
  "ask":   ["Edit(*)", "Write(*)", "Bash(*)"], // 每次询问
  "deny":  ["Bash(sudo:*)", "Bash(rm -rf:*)", "Read(**/.env)"], // 禁止
  "defaultMode": "default"                   // 未匹配时的默认行为
}

defaultMode 合法值

值含义
default标准模式，未匹配的操作按默认策略处理
acceptEdits自动接受文件编辑，其他操作仍需确认
plan仅分析规划，不执行任何修改
dontAsk不询问，按 allow/deny 规则自动处理
bypassPermissions跳过所有权限检查（仅限沙箱环境）
注意："ask" 不是合法值！如果写了 "defaultMode": "ask"， 整个 settings.json 会被 Zod schema 验证拒绝，导致所有配置（包括 env 中的 API Key）都不会被加载， 表现为 Not logged in。这是一个常见的踩坑点。

权限规则

规则含义
Read(*)允许读取任意文件
Glob(*)允许文件搜索
Edit(*)编辑文件前询问
Bash(sudo:*)禁止 sudo 命令
Bash(rm -rf:*)禁止 rm -rf
Read(**/.env)禁止读取 .env 文件

每个 CLAUDE_CONFIG_DIR 完全隔离的内容

数据说明
settings.jsonAPI Key、Base URL、Model、权限规则
.claude.json全局配置（theme、启动计数、项目配置等）
.credentials.jsonOAuth 凭据（如有）
projects/会话/对话历史
sessions/并发会话追踪
skills/ commands/自定义技能和命令
plugins/插件缓存
cache/模型能力缓存、更新日志

常见问题

Q: 两个供应商可以同时运行吗？

可以。每个实例的 lockfile、PID、配置都在各自目录下，完全独立。

Q: 新建的配置目录需要手动创建 .claude.json 吗？

不需要。首次启动时 Claude Code 会自动生成默认的 .claude.json。 只需创建 settings.json 写入 API 配置即可。

Q: 能否共享 skills 和 plugins？

不能自动共享。CLAUDE_CONFIG_DIR 是完全隔离方案。 如果需要共享，可以用软链接： 
ln -s ~/.claude/skills ~/.claude-kimi/skills
ln -s ~/.claude/plugins ~/.claude-kimi/plugins

Q: settings.json 放在哪里？

有多个位置，按优先级从低到高：

~/.claude/settings.json用户全局（或 CLAUDE_CONFIG_DIR 下）
项目根/.claude/settings.json项目共享（可入 git）
项目根/.claude/settings.local.json项目本地（gitignore）

含 API Key 的配置不要放在项目目录中（避免泄露）。

Q: 配置了 settings.json 但报 "Not logged in" 怎么办？

最常见原因：settings.json 中有字段不合法，导致整个文件被 Zod schema 拒绝。

典型错误："defaultMode": "ask"（不是合法值，应为 "default"）。

当 schema 验证失败时，整个 settings.json 被丢弃，包括 env 中的 API Key 和 Base URL，表现为未登录。

排查方法：

# 检查 settings 是否被正确加载
./cli doctor

# 或直接用环境变量注入测试（绕过 settings.json）
ANTHROPIC_AUTH_TOKEN="sk-xxx" ANTHROPIC_BASE_URL="https://..." ./cli auth status --text

# 如果环境变量方式正常，说明 settings.json 有格式问题

Q: 如何清理某个供应商配置？

rm -rf ~/.claude-kimi
